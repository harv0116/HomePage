Responsive Web Design

@media Attribute

CSS @media Attribute

Back in CSS version 2 there was a new attribute created called @media.

This let us target specific output modes for our CSS. We could create stylesheets for print only, for tv, for screen readers or for general web browsers.

<link href="main.css" rel="stylesheet" media="all" />
<link href="print.css" rel="stylesheet" media="print" />

With those two link tags added we would have a stylesheet that applies to the web pages in the browser as well as in all other situations - main.css. We also have a stylesheet that would only apply when the pages are being printed.

Print stylesheets are used to typically hide things that a user would not want to print. It can get rid of background image, remove advertisements, hide navbars and footers, or get rid of extra colours to save the end user ink.

The possible values for the old media attribute were - http://www.w3.org/TR/CSS2/media.html

If you wanted, you could put the CSS for both values inside the same CSS file.

Inside the main.css file we could keep the same content as before. Then, at the bottom of the file we would add this:

@media print{ 
          /** all the css for print styles go inside the curly braces **/ 
        }
CSS3 use of the @media attribute

When CSS3 came out it meant that a collection of new values for @media were added. These values were all aimed at deploying websites for different devices and for different sized screens.

We can still use the old values - all, screen, print, etc. but there are new ones that are more useful to us in creating layouts for different sized screens. min-width and max-width are the first two. Here is how we could use them inside a css file.

@media all and (max-width: 960px) and (min-width: 500px){
          /* This css is only applied if the screen width is between 500px and 960px */
          .wrapper{
            width: 90%;
          }
          .main{
            float:left;
            width: 60%;
          }
        }

The widths for the wrapper class and main class are only going to be applied IF the width of the screen is between those two values.

Media Queries

CSS3 Media Queries

Using the @media command with CSS3 is called a media query. We can use media queries inside the HTML link tag or inside our CSS files.

Here is the list of possible properties that we can include in a media query - http://www.w3.org/TR/css3-mediaqueries/#media1

Here are two great tutorials with graphic examples of how to use media queries to create multiple versions of the same website in order to be displayed on different sized screens.

Mobile First

Mobile First was a term coined by Luke Wroblewski - http://www.lukew.com/resources/mobile_first.asp. It was the name of a book that he wrote which encourages designers to design their media queries starting with the layout for small mobile devices. In your CSS file make the default styles ones that will be used by the small screens. The media query styles will add or alter your CSS for the larger screens.

This is now the recommended approach for "Responsive Design".

Responsive Design is the name for the technique of using multiple media queries to change elements of your design at different break points. You can use as many break points as you want. You can create a break point to change a single CSS property for one element if you need.

Using Grids in your Designs

Using Grids in your Design

When designing your web page layouts we want to make sure that things are always aligned, both vertically and horizontally. By using a grid made of columns we can measure our spacing on our designs to be consistent with our alignment.

The 960 grid system is a website that provides plugins for Photoshop that will draw the grids for you to use above or behind your layout comps. It also has a javascript bookmarklet that you can use in the browser to test the layout with your CSS and HTML. http://960.gs/ is the site. Also check out http://grids.heroku.com/ to work with grids at widths other than 960px.

The 960.gs bookmarklet lets you save a peice of Javascript as a bookmark in your browser. Then while viewing your webpage in the browser you click the book mark to run the script on your page. The script will add an overlay of the grid columns to your page as well as a control box on the left hand side that lets you change the widths of the columns and gutters as well as the line-heights and the number of columns.>

To save a bookmarklet from 960.gs simply click and drag the link to the Javascript from the page to your bookmark bar in the browser. The link on http://960.gs/ looks like a button that says "Grid Overlay Bookmark".

Flexible Grids

Responsive Design and Flexible Grids

Responsive Design is a term that refers to creating designs that adjust themselves automatically to different sized displays. Not every monitor is the same size. Not every mobile device has the same capabilities let alone screen dimensions. Android alone has over 2000 different screen sizes http://opensignal.com/reports/fragmentation-2013

.

There are three pillars to responsive design:

  1. Media Queries - used in your CSS to set properties that change at different screen sizes.
  2. Flexible Grids - using percentages for widths in your CSS layout which keep everything properly spaced and aligned at different widths.
  3. Flexible Media - using percentages to adjust images and videos so that they properly fit within your layout.

Mobile First

Mobile First

Mobile First is a term used to define an approach to responsive design. IF you want to create a series of media queries to use when designing your web pages at different widths then your default settings should be aimed at the 200px - 300px display range. Start with the smallest screen size you want to support, as your default. The term was coined by Luke Wroblewski and it is the title of his book - http://www.abookapart.com/products/mobile-first Here are a couple of articles that cover the idea of Mobile First and its pros and cons.

Understanding Viewports

Understanding Viewports

We have previously added those three <meta> tags to help mobile browsers render our webpages in a better manner. The one we need to focus on most is 

<meta name="viewport" content="width=device-width" />

This meta tag was introduced by Apple in the mobile version of Safari for the iPad. Since then it has spread to the iPhone, webkit in general and on. This page has a table that covers which browsers support this tag and which values for the content attribute - http://www.quirksmode.org/mobile/tableViewport.html You can see by looking at the chart that there are many properties and support and implementation varies greatly.

What is a Viewport

There are two basic viewports on mobile devices - Layout viewport and Visual viewport.

  • Layout viewport - this is the size of your webpage's content. Let's say that you are designing a webpage and the screen to display it was 8 feet across. You are never going to use 8 feet of space. You will set some kind of constraints on the size. Maybe your wrapper is 1200px. That is your layout viewport. You set this size in your CSS.
  • Visual viewport - This is the amount of space that the phone is telling your webpage that is available to render your webpage. In the meta tag: 
    <meta name="viewport" content="width=device-width" />
    the device-width value is the visual viewport as reported by the device. width is what you want to set the layout viewport to. The device will try to scale your webpage so that 100% of its layout viewport matches 100% of the device-width reported by the device as the visual viewport. You can set the width as an actual number but then you lose the benefit of rendering the way the device manufacturer thinks is an ideal size. He has written a series of articles about the viewports, device pixel ratios and the frustrating limitations of the device browsers. Read the first two articles. Then read them again. Media Queries max-width and min-width

When you add a media query with a max-width or a min-width you are setting test values against the width of the HTML element. The viewport meta tag will let you set the layout viewport width to a number or to the device width. That value is then compared to the media query max and min widths.

Device pixels and CSS pixels

Now, devices don't always tell the truth about their width, height or orientation.

When you set a width on an element in your CSS file you are setting the width in CSS pixels. These are not necessarily the same as device pixels. You could have a phone with 960 physical pixels. The phone is telling you that its device-width is 480 device pixels. Your CSS width for your wrapper is 1200px. You have an image in the header that is set to width: 300px.

The CSS pixels for the image are 300.

Our viewport meta tag is taking the 1200 CSS pixels for the layout viewport and scaling them to match 480 device pixels. Each of those pixels is actually two physical pixels. The device has a pixel density ratio of 2:1.

Our image is 300 CSS pixels (one quarter of the total layout width). One-quarter of the device-width is 120px. Those 120px are actually covering 240 physical pixels on the device.

As the user pinches and zooms the number of CSS pixels remains the same but your number of physical or device pixels used to display the content changes.

CSS Viewport Units

There was a new set of units that was added to CSS in 2013 that take advantage of the viewport measurements.

http://caniuse.com/#search=viewport

http://www.quirksmode.org/css/units-values/viewport.html

vw - viewport width

vh - viewport height

vmin - viewport min (use the smaller of the two measurements)

1.0vw means 1% of the viewport width.

2.8vh means 2.8% of the viewport height.
